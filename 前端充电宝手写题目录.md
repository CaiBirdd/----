# 前端手写代码高频面试题汇总（分级版）

> **说明**：本文档基于“思维导图-图例1 & 图例2”整理，并结合当前面试考察趋势进行了分级标注。
> **目标**：优先攻克 🔥 和 ⭐️ 标记的题目，这是面试的基石。

## 🏷️ 优先级说明

* 🔥 **【必会核心】**：死记硬背也要会！面试出现率极高，写不出来直接挂。
* ⭐️ **【高频重点】**：非常常见，通常用来区分候选人基础是否扎实。
* ⚙️ **【业务高频】**：实际工作中经常用到，面试官喜欢考你解决实际问题的能力。
* 👍 **【进阶/加分】**：难度稍大或考察特定原理，能写出来是很大的加分项。

---

## 1. JavaScript 基础

### 考察很多 (🔴 Legend 1)
* **手写 instanceof 方法** 🔥 `【必会核心】`
    * *考点*：原型链查找 (`proto = Object.getPrototypeOf(proto)`)。
* **手写 new 操作符** 🔥 `【必会核心】`
    * *考点*：创建对象、链接原型、绑定 this、处理返回值。
* **手写 Promise.all** ⭐️ `【高频重点】`
    * *考点*：异步并发、结果保序、错误立即返回 (Fail-fast)。
* **手写防抖函数 (Debounce)** 🔥 `【必会核心】`
    * *考点*：闭包、定时器清理、`this` 指向、立即执行版。
* **手写节流函数 (Throttle)** 🔥 `【必会核心】`
    * *考点*：时间戳版 vs 定时器版，高频事件性能优化。
* **手写 call 函数** 🔥 `【必会核心】`
    * *考点*：隐式绑定 (`context.fn = this`)、Symbol 防止属性冲突。
* **手写 apply 函数** 🔥 `【必会核心】`
    * *考点*：与 call 类似，区别在于参数处理 (数组)。
* **手写 bind 函数** ⭐️ `【高频重点】`
    * *考点*：返回新函数、柯里化传参、支持 `new` 调用（难点）。
* **实现 AJAX 请求** ⭐️ `【高频重点】`
    * *考点*：原生 `XMLHttpRequest` 状态码 (readyState, status) 处理。
* **实现深拷贝** 🔥 `【必会核心】`
    * *考点*：递归、循环引用 (`WeakMap` 缓存)、特殊对象 (`Date`, `RegExp`)。

### 考察较多 (🟠 Legend 2)
* **手写 Promise** 👍 `【进阶/加分】`
    * *考点*：状态机 (pending/fulfilled/rejected)、发布订阅模式。
* **手写 Promise.then** 👍 `【进阶/加分】`
    * *考点*：链式调用（返回新 Promise）、微任务 (`queueMicrotask` / `setTimeout`)。
* **手写 Promise.race** ⚙️ `【业务高频】`
    * *考点*：谁快用谁，常用于请求超时控制。
* **手写类型判断函数** ⚙️ `【业务高频】`
    * *考点*：`Object.prototype.toString.call(val)` 的封装。
* **函数柯里化的实现** ⭐️ `【高频重点】`
    * *考点*：闭包参数收集、`fn.length` 判断参数个数。

---

## 2. 数据处理

### 考察很多 (🔴 Legend 1)
* **实现数组的乱序输出** ⭐️ `【高频重点】`
    * *考点*：Fisher-Yates 洗牌算法，保证随机性均匀。
* **实现数组的扁平化 (Flatten)** 🔥 `【必会核心】`
    * *考点*：递归、`reduce`、栈迭代、`ES6 flat`。
* **实现数组去重** 🔥 `【必会核心】`
    * *考点*：`Set` (最简)、`Map`、`indexOf`、对象键值对。
* **将数字每千分位用逗号隔开** ⚙️ `【业务高频】`
    * *考点*：正则零宽断言 (`/\B(?=(\d{3})+(?!\d))/g`) 或 `toLocaleString`。
* **实现非负大整数相加** ⭐️ `【高频重点】`
    * *考点*：大厂常考，解决 JS `Number.MAX_SAFE_INTEGER` 溢出问题，模拟竖式加法。
* **实现 add(1)(2)(3)** ⭐️ `【高频重点】`
    * *考点*：柯里化变形、重写 `toString` 或 `valueOf` 实现隐式转换。
* **使用 ES5 和 ES6 求函数参数的和** ⚙️ `【业务高频】`
    * *考点*：ES5 `arguments` (类数组转数组) vs ES6 `...rest`。
* **解析 URL Params 为对象** 🔥 `【必会核心】`
    * *考点*：正则匹配或 `split` 分割，处理解码 (`decodeURIComponent`)。

### 考察较多 (🟠 Legend 2)
* **实现日期格式化函数** ⚙️ `【业务高频】`
    * *考点*：`Date` API 方法，字符串补零逻辑。
* **实现数组元素求和** ⚙️ `【业务高频】`
    * *考点*：循环或 `reduce` 累加。
* **使用 reduce 求和** ⚙️ `【业务高频】`
    * *考点*：`reduce` 的基本用法。
* **将 js 对象转化为树形结构** 🔥 `【必会核心】`
    * *考点*：**Vue 业务必考**。处理扁平数据 (id, pid) 转 Tree，利用 Map 优化性能。

---

## 3. 场景应用

### 考察很多 (🔴 Legend 1)
* **循环打印红黄绿** ⭐️ `【高频重点】`
    * *考点*：异步流程控制。Promise 链式调用 (`.then().then()`) 或 `async/await`。

### 考察较多 (🟠 Legend 2)
* **小孩报数问题** 👍 `【进阶/加分】`
    * *考点*：约瑟夫环问题，数组操作或循环链表模拟。
* **查找文章中出现频率最高的单词** ⚙️ `【业务高频】`
    * *考点*：字符串处理 + 哈希表 (`Map`/`Object`) 统计词频。
* **实现双向数据绑定** 🔥 `【必会核心】`
    * *考点*：**Vue 面试必问**。`Object.defineProperty` (Vue2) 或 `Proxy` (Vue3)。
* **实现斐波那契数列** ⭐️ `【高频重点】`
    * *考点*：递归的性能陷阱、缓存优化、动态规划 (迭代法)。
* **使用 setTimeout 实现 setInterval** ⭐️ `【高频重点】`
    * *考点*：递归调用 `setTimeout`，理解 Event Loop 和 Timer 漂移。
* **实现 jsonp** 👍 `【进阶/加分】`
    * *考点*：跨域原理，动态创建 `<script>` 标签。
* **判断对象是否存在循环引用** 👍 `【进阶/加分】`
    * *考点*：递归遍历 + `Map` / `WeakMap` 记录已访问对象。

## 💡 给 Vue 开发者的刷题建议

1.  **第一阶段（基础夯实）**：
    优先搞定所有带有 🔥 **【必会核心】** 标签的题目。特别是 `深拷贝`、`防抖节流`、`数组去重/扁平化`，这些是面试的门票。

2.  **第二阶段（框架关联）**：
    重点攻克以下三个，因为它们直接关系到你对 Vue 原理的理解：
    * **实现双向数据绑定** (Vue 响应式核心)
    * **将 JS 对象转化为树形结构** (Vue 组件树/路由菜单常用)
    * **手写 Promise.all** (前端并发请求常用)

3.  **第三阶段（算法补充）**：
    最后复习 **大整数相加** 和 **数组乱序**，这两道题在稍微大一点的公司（或者初创公司想考算法时）非常容易遇到。
